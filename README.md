# NoteNet

Описание
-----------------
NoteNet е учебен уеб-проект (Django), който управлява материали, класове, дисциплини и потребителски профили.

Основни функционалности
- Управление на материали и файлове
- Профили на потребители
- Дисциплини и предмети

Prerequisites / Предпоставки
- Python 3.11+ (препоръчително) или съвместима 3.x инсталация
- Git

Инсталиране и стартиране (локално)
-----------------
1. Клонирайте репото и влезте в директорията на проекта:

```bash
git clone <repo-url>
cd NoteNet/src
```

2. Създайте виртуална среда и я активирайте:

```bash
python3 -m venv venv (python за windows)
source venv/bin/activate
```

3. Инсталирайте зависимостите. Ако имате `requirements.txt` използвайте:

```bash
pip install -r requirements.txt
```

Ако нямате `requirements.txt`, инсталирайте основните пакети:

```bash
pip install Django==6.0.3 python-dotenv
# Ако използвате Postgres локално:
pip install psycopg2-binary
```

4. Конфигурация на база данни

- По подразбиране проектът ще използва SQLite ако променливата `DB_PASSWORD` не е зададена.
- Ако искате да използвате PostgreSQL, създайте `.env` в корена на проекта (там където е `src`) и добавете:

```bash
DB_PASSWORD=your_postgres_password
# Други env променливи по нужда
```

5. Приложете миграции и създайте суперпотребител (ако е нужно):

```bash
cd src
python3 manage.py migrate
python3 manage.py createsuperuser
```

6. Стартирайте development сървъра:

```bash
python3 manage.py runserver
```

Тестове
-----------------
Проектът използва `pytest` (виж `pyproject.toml`). Стартирайте тестовете така:

```bash
pytest
```

Често срещани проблеми
-----------------
- "no such table: django_session": Стартирайте `python3 manage.py migrate` за да създадете необходимите таблици.
- Проблеми с достъп до Postgres: уверете се, че Postgres работи и `DB_PASSWORD` е коректно зададен.

Допълнителни бележки
-----------------
- Файловете за статични/медийни ресурси са в `static/` и `media/`.
- Конфигурацията на проекта е в `src/notenet_project/settings.py`.

Dependencies (кратко описание)
-----------------
- `asgiref` — предоставя помощни библиотеки за ASGI и асинхронни операции в Django.
- `beautifulsoup4` — парсва HTML и XML, улеснявайки извличането на данни от уебстраници.
- `certifi` — пакет с актуални доверени коренни SSL сертификати.
- `charset-normalizer` — открива и декодира текстови енкодинги в отговори и файлове.
- `Django` — основният уеб фреймуърк, върху който е изграден проектът.
- `idna` — поддържа кодирани международни домейн имена (IDNA).
- `psycopg2-binary` — драйвер за връзка между Django и PostgreSQL база данни.
- `python-dotenv` — зарежда променливи на средата от `.env` файлове.
- `requests` — лесен HTTP клиент за изпращане на заявки към външни услуги.
- `soupsieve` — осигурява CSS селектори за `beautifulsoup4` при намиране на елементи.
- `sqlparse` — формат и парсинг на SQL заявки, полезен при инспекция на SQL.
- `typing_extensions` — предоставя допълнителни типови подсказки и разширения за Python typing.
- `urllib3` — ниско ниво HTTP библиотека, използвана от `requests`.
